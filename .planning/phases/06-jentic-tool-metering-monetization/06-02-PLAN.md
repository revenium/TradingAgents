<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM -> tool arg | `query` string flows from the model into Jentic's HTTP API |
| Jentic external API -> agent | `ExecuteResponse.output` (untrusted third-party content) is returned to the LLM as a tool result |
| tool -> Revenium metering | tool event crosses to /v2/tool/events; must never block the run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02 | Denial of Service | metering / Jentic call inside the graph | mitigate | fail-soft: any Jentic error returns NO_DATA_AVAILABLE; @meter_tool is fail-open; the run never crashes (tested) |
| T-06-03 | Tampering (prompt injection) | ExecuteResponse.output returned to LLM | accept | output is passed through LangChain's normal tool-result channel (not eval'd); no elevated action; same trust model as existing get_news; documented |
| T-06-01 | Information Disclosure | JENTIC_AGENT_API_KEY | mitigate | key read from config only, lazy-imported Jentic, never included in warning logs; log symbolic reason strings only |
| T-06-04 | Spoofing (pricing mismatch) | emitted toolId | mitigate | toolId sourced from jentic_tool_id default `jentic:news`; identical string reused for ToolResource registration in 06-03 (L6) |
</threat_model>
