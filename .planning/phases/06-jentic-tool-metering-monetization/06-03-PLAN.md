## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator -> Revenium Platform API | rev_sk_ write key registers a priced ToolResource over HTTPS |
| config -> pricing host | revenium_profitstream_url selects which host receives the tool registration |
| Jentic external API -> live run | real third-party news content enters the pipeline during live verify |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Spoofing (pricing mismatch) | toolId in event vs ToolResource | mitigate | both sourced from jentic_tool_id `jentic:news`; checkpoint step 4 verifies non-zero cost proves the match (L6) |
| T-06-05 | Elevation/Config error | pricing host (api.revenium.ai vs api.prod.ai.hcapp.io) | mitigate | host is config-driven (revenium_profitstream_url), not hardcoded; script hints the alternate host on 401/404; operator confirms (Open Question 1) |
| T-06-01 | Information Disclosure | rev_sk_ + jentic keys in scripts | mitigate | keys read from env/config; never printed by either script; validate_billing skip-pattern reused |
| T-06-02 | Denial of Service | live execute in the run | mitigate | fail-soft tool (06-02) returns sentinel; live run never crashes on a provider hiccup |
</threat_model>
