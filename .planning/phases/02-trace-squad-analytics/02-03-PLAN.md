<objective>
Close GAP-02-01 (BLOCKING, TRC-01/TRC-02) and unblock GAP-02-02 (TRC-03). A live `propagate()` run currently produces 20 spans with an empty/inconsistent `trace_id` and 0/19 post-first spans carrying `parent_transaction_id` — no single grouped trace, no parent-arrow dependency tree. Root cause (reproduced keyless in 02-VERIFICATION.md): LangGraph executes every node inside its own `copy_context().run(...)`, so the `current_parent_transaction_id.set()` performed in node A's `on_llm_end` lands in node A's private context copy and is invisible to node B's `on_chat_model_start`. The Phase 2 research premise ("synchronous main thread ⇒ contextvar visible to next node") is false cross-node.

This is one thin end-to-end slice: move run-scoped trace state off per-node contextvars and onto the single shared handler instance (one `ReveniumCallbackHandler` serves the whole run), prove it keyless with a cross-node regression test, then re-validate live with the existing gate.

Purpose: Restore the single-trace + parent-chain structure that TRC-01/TRC-02/TRC-03 depend on, and lock the regression in keyless.
Output: Handler-instance run-scoped state (`_last_transaction_id`, `begin_run`/`end_run`); contextvar fallback retained; cross-node keyless test; live re-validation pass.
</objective>
