<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process → Revenium Jobs/Outcomes API | `create_job`/`report_outcome` HTTP writes carry an x-api-key and revenue value across the network |
| deployment env → config | `REVENIUM_SIGNAL_PRICE` / billing key / host arrive via env vars (operator-controlled) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | Information Disclosure | billing emit + validate_billing.py logging | mitigate | Log only trace_id, ticker, signal_price, and result codes; never the api key or raw response body (enforced in billing.py docstring + symbolic logger args). |
| T-04-04 | Denial of Service | billing emit blocking a run | mitigate | All emits are fail-open on a daemon thread; an unreachable host or timeout is swallowed (warn) and never blocks/crashes the caller. `enabled is False` when no key → total no-op. |
| T-04-05 | Tampering | signal price via env var | accept | `TRADINGAGENTS_SIGNAL_PRICE` is operator/deployment-controlled, not runtime user input; ASVS L1 requires no extra validation (float coercion already applied). |
| T-04-SC | Tampering | pip installs | accept | No new packages; `AgenticOutcomeClient` already vendored in `uv.lock`. No package-legitimacy checkpoint required. |
</threat_model>
