<objective>
Deliver the in-app live cost panel (CLI-01, CLI-02). A dedicated Rich panel inside the existing `Live` layout shows per-agent dollar cost (primary) and tokens (secondary), highlights the priciest agent as the hotspot, and collapses debate-loop agents to one `×N` row with summed cost — all fed from the callback handler's `agent_costs` dict (no parallel accumulator).

This is a complete vertical slice: after this plan, an operator running a live ticker sees "where the money goes" in-app without opening Revenium.

Purpose: Make the debate-loop cost hotspot pop on stage during the FCAT demo (the CLI-02 "aha" visual).
Output: `pricing.py` (local cost lookup), an extended `agent_costs` schema, and the live `_build_cost_panel` wired into the CLI layout.

Implements decisions: D-01 (dedicated panel), D-02 ($ primary / tokens secondary), D-03 (hotspot highlight), D-04 (feed from `agent_costs`), D-05 (×N collapse), D-06 (applies to debate agents only).
</objective>
